# MAGI-CONTENT 改进路线

MAGI-CONTENT 是内容搬运流水线（粘链接→下载→转写/OCR→AI 摘要），FastAPI/uvicorn 单进程 + ThreadPoolExecutor(max_workers=3) + SQLite。运行于 Python 3.12（`config.py` 模块级 `Config | None` 需 3.10+；本机系统 python 是 3.9 无法整栈跑）。

改进路线分三波（2026-06 规划）：
- **第一波（稳采集）—— 已完成**：修复 `downloader.py` 对所有平台固定用 B站 Referer 的 bug；新增 `fingerprint.py`（UA 池轮换 + 按平台 Referer + `user_agent_override`）、`throttle.py`（按平台节流，间隔读 config）；启用 `PlatformRule.alt_downloader` 失败降级链；`xhs_cookie.py` 加 `validate_xhs_cookie` 结构性健康预警；`config.py` 外置 `proxy_pool / rate_limit_*_sec / user_agent_override`。经对抗式审查修了 3 个回归（BaseException 吞中断、节流后缺取消复查、节流槽不回滚已文档化）。
- **第三波（强处理）—— 核心已建（2026-06）**：摘要引擎做成**可插拔**（`app/services/engines/`：deepseek/claude/qwen/minimax，注册表 + `SUMMARY_ENGINE` 选择，不预设默认=deepseek保现状）；`summarizer.py` 重构为**引擎 + 长文本 map-reduce（消灭 30 万字硬截断）+ 多模态视觉**（图文且引擎 supports_vision 时直接读图，否则 OCR 文本）；`summary_format.py` 扩展结构化字段（章节时间轴/实体/行动项/标签，向后兼容旧摘要）；**向量检索** `vectorstore.py`（sqlite-vec + 可插拔 embedder：`EMBEDDING_BACKEND=openai/local/空`，pipeline 完成后建索引，`/api/search/semantic` 端点）；**X(Twitter)** 提为一等公民平台规则。前端 `markdown_util.py` + style.css 把新字段渲染为小节标题 + 标签 chips。全部在 3.12 venv 单元验证通过（map-reduce 7段+1归并不丢尾、视觉分流、X 检测、向量 KNN）。引擎/embedder 真实调用未测（无可用 key：DeepSeek key 是占位符）。
- **关键环境**：已 `brew install python@3.12`（3.12.13）；`.venv`（3.12，全量依赖安装中）；`.venv-bench`（3.9，引擎对比基准台 `scripts/bench/`）。测试方法论：自取公开 URL + 本地 whisper 转录 + **子代理扮演 LLM**（零付费 key 跑 E2E）。
- **第三波收尾**：✅ 前端语义搜索框（首页「语义检索」按钮 + `/api/search/semantic`，实运行验证）。
- **E2E 真实数据**：✅ 跑通两条路径（零付费 key）——YouTube(yt-dlp)「Me at the zoo」+ B站(yutto, 无 cookie)「BV1ALEg6UEeN」，均 真实下载→ffmpeg 提取→本地 Whisper 转录→子代理(我)扮 LLM 出结构化摘要→渲染。编排脚本 `scripts/e2e/run_dl_transcribe.py`。**注意**：whisper-small 对 B站 噪杂 meme 音频转录质量差（输出garbled），高质量需大模型或云端听悟后端——是模型/内容问题，非流水线 bug。
- **第二波（扩平台）基本完成**：✅ `downloader.py` 插件化（`DownloadContext` + `DOWNLOAD_PLUGINS` 注册表 + `register_downloader` 扩展点，降级链/节流不变）；✅ 新增 微博/快手/Instagram/TikTok/X + **local（本地文件）** 平台规则 + Referer（现 11 个非 generic 平台）；微博/快手/IG 配 alt=gallerydl；✅ **本地文件上传**：`local` 下载器插件（按扩展名→video_path/images_dir）+ `platform_detector` 识别本地路径/file:// + `POST /api/jobs/upload` 端点 + 前端上传面板。**全部真实 E2E 验证**：本地 mp4 → 跳过下载 → 提取 → Whisper → 真 DeepSeek 摘要。待办：微博图文/快手可接更专门下载器（插件加一行即可）。
- **真引擎实测（DeepSeek）✅**：用户给了可用 DEEPSEEK_API_KEY（已写入 .env）。验证：真实摘要生产路径 OK；12.8 万字 map-reduce 切 2 块、首尾均不丢（旧代码会超上下文/截断）。

  - **xhigh 代码审查 + 修复（2026-06-15）**：对本次改动跑 9 角度审查（71 代理，54 候选→15 findings），全是新代码引入。已全部修复+实测：① 🔴`wechat_downloader._fetch_html` 原用默认 urllib opener 抓任意 URL→SSRF+file:// LFI（实测可读 /etc/passwd）：改为仅 http/https + 手动 OpenerDirector 排除 File/FTP handler + 禁重定向 + 响应体 10MB 上限；私网/保留 IP 仅在**直连**时拦（配 http 代理时跳过，避免代理/NAT 误伤）。② api.py `create_job` 加 downloader 白名单。③ `subtitles.find_subtitle_file` 语种由子串改**精确**匹配（原 `zh` 吃掉 `zh-Hans`），简体优先。④ `parse_subtitle_text`：头部元数据(WEBVTT/NOTE/Language:)只在首条时间轴前算噪声(不再误删正文台词)；去重改为**仅相邻完全重复**(保留副歌等合法重复)。⑤ 字幕复用加开关 `SUBTITLE_REUSE`(默认开)+`SUBTITLE_USE_AUTOCAPTION`(默认**关**，自动机翻字幕质量差且会串语种、绕过转写后端)。⑥ wechat 跳过 socks 代理(urllib 不支持)。低优先级未改：wechat 阻塞抓取不响应取消(最多等 30s timeout，可接受)。17 测试通过。
